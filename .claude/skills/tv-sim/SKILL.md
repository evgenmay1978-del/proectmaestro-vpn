# Sim-first для ТВ-экрана

⛔ Правило проекта: раскладку ТВ НЕ менять вслепую — сначала отрисовать симуляцию по ЧИСЛАМ ИЗ KOTLIN (не из спеки!) и посмотреть глазами (Read на PNG).

1. Числа: `app/src/main/java/com/maestrovpn/tv/compose/screen/tvhome/TvEskizSpec.kt` (арт-пространство 1672×941; s = 960/1672 dp на арт-px; эталон экрана 960×540dp).
2. Рендер: python из venv `/root/vpn_bot/venv/bin/python3` (PIL+numpy стоят) — композитишь `app/src/main/res/drawable-nodpi/tv_bg_{off,on}.webp` + прямоугольники/текст на предлагаемых позициях, направляющие линии (штрих кольца: OFF до y=773, ON до y=830 — замерено по пикселям). Сохраняй в scratchpad, смотри через Read.
3. Полный пайплайн ассетов (перегенерация кропов/фонов из /root/vpnon.png|vpnoff.png): `ops/tv-eskiz-pipeline.py` (OUT-путь поправить на свой scratchpad). Фоны экспортируются LOSSLESS — не понижать.
4. Правила: кропы НЕ масштабировать на фокусе; фокус = чёткая рамка (никаких blur-glow/state-layer); арт-битмапы с FilterQuality.High; статус-блок обязан влезать в 788..936 арт-y в ОБОИХ состояниях.
5. После кода: CI-компайл через dispatch android-test.yml (локального SDK нет), merge в main = авторелиз (см. CLAUDE.md).
