# Sim-first для ТВ-экрана (v4 «материал мобильной версии», 2026-07-12)

⛔ Правило проекта: раскладку ТВ НЕ менять вслепую — сначала отрисовать симуляцию по ЧИСЛАМ ИЗ KOTLIN (не из спеки!) и посмотреть глазами (Read на PNG).

1. Числа: `app/src/main/java/com/maestrovpn/tv/compose/screen/tvhome/TvEskizSpec.kt` — арт-пространство **1920×1080**, s = экран-dp/1920 (эталон 960×540dp). Панели = INNER-прямоугольники + поля M=28 (тень запечена в ассет): ассет кладётся в (x−M, y−M), размер (w+2M)×(h+2M).
2. Рендер: python из venv `/root/vpn_bot/venv/bin/python3` (PIL+numpy) — композитишь `app/src/main/res/drawable-nodpi/tvm_bg_{off,on}.webp` + панели `tvm_*.webp` на позициях спеки + текст/иконки. Сохраняй в scratchpad, смотри через Read.
3. Полный пайплайн ассетов (перегенерация из ТЕЛЕФОННОГО арта репо — home_backdrop*, frame_bar/frame_button/frame_qr): `ops/tv-mobile-kit.py`. Все выходы LOSSLESS — не понижать. Числа генератора и TvEskizSpec ОБЯЗАНЫ совпадать.
4. Правила: растр НЕ растягивать и НЕ масштабировать на фокусе; фокус = чёткий янтарный двойной контур (никаких blur-glow/state-layer); арт-битмапы с FilterQuality.High; скролла на ТВ-хоуме НЕТ — всё в один экран 1920×1080; статус (красный OFF / зелёный ON, как на телефоне) + аккаунт-бар живут в левой зоне под медальоном (орб-центр 422,600, r_vis 176).
5. После кода: CI-компайл через dispatch android-test.yml (локального SDK нет), merge в main = авторелиз (см. CLAUDE.md).
